## Scoring Legend

- **Reach (R):** Users/month impacted (1-10 scale, where 10 = every active user)
- **Impact (I):** 0.25 = minimal, 0.5 = low, 1 = medium, 2 = high, 3 = massive
- **Confidence (C):** 0.5 = low (gut feel), 0.8 = medium (some data), 1.0 = high (measured)
- **Effort (E):** Person-weeks to build (0.5 = few hours, 1 = 1 week, 4 = 1 month, 8 = 2 months)
- **RICE Score** = (R x I x C) / E

## Notes

### Scoring Rationale

- **#1 scores highest (60.0)** because it affects every user (R=10), has massive impact on conversion (I=3), is validated by the 71% drop-off data (C=1.0), and takes only a few hours (E=0.5). Already implemented — validate with analytics.
- **#4 (credit packs) scores 16.0** because credit packs already exist in code (`CREDIT_PACKS` in pricing.tsx with 5/10/25/50/100 tiers). What is missing is upsell prompts at the right moments: post-analysis when credits are low, in workspace dashboard, and in email sequences. Mostly copy + placement work.
- **#5 (onboarding wizard) scores 9.60** because first-time experience directly drives activation rate. At 9 out of 10 users seeing it, 2x impact, and only 1.5 weeks of work, this is the highest-leverage P1 feature.
- **Revenue features cluster around 3-6 RICE** because paying users are a smaller subset (lower R) but each conversion has outsized impact (higher I). Annual billing (#15) is a quick win because Stripe subscription switching is straightforward.
- **Growth features (#6 embed widget, #13 referral)** score well because they create acquisition channels that compound over time. The embed widget (#6) is particularly efficient — one implementation, distributed across many partner sites.
- **AI features score moderately** due to higher effort and lower confidence. The weekly AI email (#9) is the exception — personalised outbound has proven engagement lift and can reuse existing SVI scoring + evidence data.
- **Phase 4+ features score lowest** primarily due to higher effort and lower confidence at this stage. Re-score these when Phase 3 completes and we have real user data.

### Re-scoring Triggers

Re-score the backlog when:
1. **Analytics baseline established** (after #2 ships) — real Reach data replaces estimates
2. **Phase 2 complete** — Phase 3 features get updated Confidence scores
3. **Any feature takes 2x estimated Effort** — recalibrate similar features
4. **User feedback contradicts Impact assumptions** — survey Founding 50 cohort
5. **Revenue milestones hit** — $5K MRR triggers re-prioritisation of Enterprise/Team features
6. **Partnership signed** — accelerator deal triggers immediate re-score of #6, #33, #39
