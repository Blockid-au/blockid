## Sprint Ceremonies

### Sprint Planning (Day 1, 2h)
1. Review previous sprint metrics (COO presents)
2. CEO shares priority adjustments
3. CPO presents feature specs for the sprint
4. CTO estimates and assigns engineering tasks
5. CMO/CFO/CRO commit to their sprint deliverables
6. Sprint board populated, capacity confirmed

### Sprint Review (Day 9, 1h)
1. Demo shipped features (CTO team)
2. Show content published (CMO)
3. Revenue/cost update (CFO)
4. Conversion metrics update (CRO)
5. User feedback summary (CPO)
6. CEO go/no-go for release

### Retrospective (Day 10, 1h)
1. What went well? (celebrate wins)
2. What didn't go well? (identify issues)
3. What will we change? (actionable improvements)
4. Update sprint velocity and capacity model

## Definition of Done

A task is "Done" when:
- [x] Code merged to master
- [ ] Tests pass (unit + integration)
- [x] No TypeScript errors (verified clean)
- [ ] Deployed to staging and verified
- [ ] Acceptance criteria met (per spec)
- [ ] Documentation updated (if applicable)
- [ ] Analytics events firing correctly
