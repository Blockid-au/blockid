# CPO Goals — Chief Product Officer

## Mission
Define what BlockID.au builds and why. Own product strategy, user experience, feature prioritization, and product analytics. Ensure every feature solves a real founder pain point and drives measurable engagement.

## Roadmap Ownership

| Phase | CPO Responsibility |
|-------|-------------------|
| Phase 1 (Idea) | SVI analysis UX, report design (DONE) |
| Phase 2 (Validation) | Evidence vault UX, badge system, living reports |
| Phase 3 (MVP) | Multi-project dashboard, data room UX, valuation display |
| Phase 4 (Equity) | Cap table UI, equity split experience, vesting visualization |
| Phase 5 (Tokenization) | Token dashboard, on-chain activity display |
| Phase 6 (Investment) | Investor view, shareable reports, pitch deck integration |

## Responsibilities

1. **Product Strategy** — Define quarterly product themes aligned with phase roadmap
2. **User Research** — Conduct founder interviews, usability tests, pain point discovery
3. **Feature Prioritization** — RICE scoring, impact/effort matrix, backlog management
4. **UX Design** — Information architecture, user flows, interaction design
5. **Design System** — Maintain and evolve BlockID design system (colors, components, patterns)
6. **Product Analytics** — Feature adoption tracking, cohort analysis, retention curves
7. **A/B Testing** — Hypothesis-driven experimentation on key flows
8. **Competitive Product Analysis** — Feature comparison with Carta, Pulley, Cake Equity
